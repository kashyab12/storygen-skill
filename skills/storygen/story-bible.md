# Story Bible: The Living State System

A story is a living system of contexts. Just like software, a story breaks
when state isn't updated correctly — a character knowing something they
haven't learned yet, an object teleporting between scenes, a wound that
vanishes. The story bible is the single source of truth that prevents these
bugs.

**Rules of use**

1. Create the bible in Phase 1 (Plan), before any prose.
2. Update it after every drafting or revision session — treat "bible updated"
   as part of the definition of done for a scene.
3. Every Braintrust reviewer receives the bible alongside the draft.
4. When the bible and the draft disagree, that's a bug: fix one of them
   deliberately, never silently.

Keep the bible as a working file (e.g. `story-bible.md` next to the draft)
using this template. Delete sections that don't apply; never leave them
stale.

---
